# acceptance criteria

## AC-01 ホーム表示

- 添付イメージに準拠したカード構成で表示される
- `今日のログ`、本日の本数、経過時間、今月の金額、履歴、ナビバーが見える

## AC-02 喫煙登録

- `+` ボタン1回で本日の本数が `+1` される
- 履歴先頭に最新時刻が追加される
- Homeの`+`で作成されたレコードは `quantity=1` で保存される

## AC-02b 本数保持

- レコードに `quantity` が保存される
- 当日本数はレコード件数ではなく `quantity` 合計で表示される

## AC-02c 将来一括登録の前提

- `trendEligible=false` のレコードを保存できる
- `trendEligible=false` は将来のトレンド計算対象外として扱える構造である

## AC-03 経過時間

- 最新登録からの経過時間が `hh:mm:ss` で表示される
- 1秒ごとに更新される

## AC-04 トレンド矢印

- 本日の本数カード右側に矢印が表示される
- 過去7日同時刻平均と比較して `↑ / → / ↓` が正しく切り替わる

## AC-05 今月金額

- 1箱金額設定を反映して今月金額が計算される
- 計算結果は3桁カンマ区切りで表示される

## AC-06 履歴制限

- 履歴は降順で表示される
- 11件以上あっても表示は10件まで

## AC-07 ナビバー

- 画面下部に `Home / Analysis / Trend / Settings` が表示される
- Homeがアクティブ表示される

## AC-08 永続化

- 再読み込み後も記録と設定（1箱金額）が保持される
