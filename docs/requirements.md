# requirements

## 1. 目的

禁煙アプリとして、日々の記録に加えて過去データを手動管理できるようにする。  
対象は個人利用の iPhone PWA。

## 2. 対象画面

- Home（既存）
- Data（今回追加）
- Settings（既存）
- Analysis（未実装）

## 3. Dataタブ要件

### R-D01 タブ名称

- `Trend` を `Data` に変更する

### R-D02 役割

- Dataタブは「過去データ管理」を目的とする

### R-D03 データ操作

- 選択日のデータに対して以下を行える
  - 追加
  - 一括追加
  - 変更
  - 削除

### R-D04 カレンダー

- 画面上部に月表示カレンダーを表示する
- 前月/次月へ移動できる
- デフォルト選択日は当日
- カレンダーで選択した日のデータを表示する

### R-D05 当日デフォルト

- Dataタブを開いた時点で当日を選択状態にする

### R-D06 選択日データ表示

- 画面下部に選択日のレコード一覧を表示する
- 表示項目:
  - 時刻
  - 本数
  - 分析に使うか（trendEligible）

### R-D07 追加ボタン

- `追加` ボタンを設置する（1件追加）
- `一括追加` ボタンを設置する（複数本まとめて追加）

## 4. 既存要件との整合

- Homeの `+` は従来どおり1本固定
- 一括追加データは `trendEligible=false` を基本に登録する
- 日次本数・今月金額には quantity を反映する

## 5. 非機能要件

- Data一覧は画面内スクロール可能
- Service Worker登録失敗時もアプリ起動継続
