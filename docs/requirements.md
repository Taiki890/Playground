# requirements

## 1. 目的

喫煙記録を漏れなく残し、日次進捗・金額・傾向を確認できるようにする。  
対象は個人利用の iPhone PWA。

## 2. 画面方針

- 画面イメージは `iPhone 17 - 1.png` を基準にする
- 色は可読性を優先し、必要ならデザイン調整する
- 主操作は片手1タップで完了する

## 3. 機能要求（今回）

### R-01 Home表示

- 今日のログ、本日の本数、経過時間、今月の金額、履歴、下部ナビバーを表示する

### R-02 記録登録（Home）

- Homeの `+` ボタンは常に1本登録のみ
- Homeからは本数指定登録はしない

### R-03 記録データ構造

- 記録は日時だけでなく本数を保持する
- 基本は `quantity=1`
- 将来の一括登録に備え、トレンド分析対象かどうかを保持する

### R-04 一括登録の扱い（将来機能の前提）

- 一括登録レコードは日次本数・今月金額に反映する
- 一括登録レコードはトレンド分析には使わない
- 理由: 実際の喫煙時刻が不正確なため

### R-05 トレンド表示

- 本日の本数右側に `↑ / → / ↓` を表示
- 比較基準は過去傾向（詳細はspec）
- トレンド計算は `trendEligible=true` のレコードのみを対象

### R-06 金額計算

- 1箱金額を設定できる
- 今月金額を算出する（1箱20本）

### R-07 履歴

- 履歴は新しい順に最大10件表示

## 4. 非機能要求

- iPhone Safariのホーム画面追加で起動できること
- 再読み込み後も記録が保持されること
- オフラインで保存済みデータが参照できること

## 5. スコープ外

- Analysis詳細要件
- Trend詳細要件
- Settingsの一括登録UI
