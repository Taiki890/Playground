# requirements

## 1. 目的

喫煙記録を継続しやすくし、今日の進捗・同時刻比較トレンド・今月コストを1画面で把握できるようにする。  
対象は個人利用の iPhone PWA。

## 2. 対象画面

- Home（実装済み）
- Settings（実装済み: 1箱金額設定 / データ入出力）
- Analysis / Trend（未実装、後続要件）

## 3. 機能要求

### R-01 Home表示

- 今日のログを表示する
- 本日の本数を表示する（quantity合計）
- トレンド矢印を表示する（↑ / → / ↓）
- トレンドで使用した現在時刻基準の平均値を矢印の横に表示する（例: `avg 2.4`）
- 経過時間を `hh:mm:ss` で表示し、1秒ごとに更新する
- 今月の金額を表示する（3桁カンマ）
- 履歴を最新順で最大10件表示する

### R-02 記録登録

- Homeの `+` ボタンは1タップで1本登録する
- Homeの `+` では本数指定は不可
- 登録後に本数・経過時間・履歴・今月金額・トレンドを更新する

### R-03 レコード構造

- 各レコードに以下を保持する
  - `createdAt`
  - `dayKey`
  - `quantity`
  - `trendEligible`

### R-04 一括登録の前提

- 将来の一括登録では `quantity >= 1` を保持できること
- 一括登録は日次本数と今月金額に反映すること
- 一括登録はトレンド分析から除外できること（`trendEligible=false`）

### R-05 Settings

- 1箱あたり金額（円）を保存できること
- 全データをJSONでエクスポートできること
- JSONからインポートして復元できること

### R-06 ナビゲーション

- 下部ナビバーに `Home / Analysis / Trend / Settings` を表示すること
- 現在は Home と Settings のみ遷移可能

## 4. UX要求

- Homeは原則1画面表示で操作可能
- 履歴エリアのみ内部スクロール可能
- `+` ボタンは押しやすい横長サイズ

## 5. 非機能要求

- iPhone Safariのホーム画面追加で起動可能
- `service worker` 登録に失敗しても通常UIは起動継続
- 再読み込み後も記録・設定を保持する

## 6. スコープ外

- Analysis 詳細要件
- Trend 詳細要件
- 一括登録UI
